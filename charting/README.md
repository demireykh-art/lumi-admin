# 피부과 전용 AI 보이스 차팅 (MVP)

진료 대화를 녹음 → 음성인식(STT) → 피부과 전용 프롬프트로 **SOAP 차트** 생성 → 편집 후
**원클릭 복사**해 기존 EMR에 붙여넣는 데스크톱 위젯입니다.

> **이건 "받아적는 AI"입니다.** 의학적 판단은 의사가 하고, AI는 말한 내용을 서식에 맞게
> 재배열만 합니다. 언급되지 않은 내용은 지어내지 않고 "언급 없음"으로 표기하도록 프롬프트에
> 명시되어 있습니다. (진단/추론용 자체 의료 엔진은 이 라이트 버전에 필요하지 않습니다.)

## 구성

| 파일 | 역할 |
|---|---|
| `index.html` / `styles.css` | 플로팅 위젯 UI |
| `app.js` | 녹음 · STT · 비식별화 · LLM 차팅 · 클립보드 복사 로직 |
| `config.js` | **피부과 용어 사전 + SOAP/상담 프롬프트 + 기본 설정** (여기서 튜닝) |
| `main.js` / `preload.js` | Electron 래퍼 — 항상 위·프레임 없는·드래그 이동 플로팅 창 |
| `package.json` | Electron 실행/빌드 매니페스트 |

## 실행 방법

### 1) 빠른 테스트 — 웹 브라우저
설치 없이 확인할 수 있습니다. Chrome/Edge에서 `index.html`을 여세요.
(마이크·클립보드 접근은 `https` 또는 `localhost`에서 동작합니다. 로컬 서버 예:
`npx serve charting` 후 표시되는 주소 접속.) 브라우저에서는 최소화/닫기 버튼은 숨겨집니다.

### 2) 실제 사용 — Electron 플로팅 위젯 (Windows 우선)
```bash
cd charting
npm install
npm start           # 항상 위에 뜨는 작은 위젯 실행
npm run dist:win    # Windows 설치 파일(.exe) 빌드
```
헤더를 드래그해 위치를 옮기고, 진료 화면 구석에 상주시킨 뒤 EMR 위에서 그대로 사용합니다.

## 최초 설정 (⚙ 설정)

1. **STT** — Base URL / 모델 / API Key. 기본값은 OpenAI 호환 Whisper(`whisper-1`).
2. **LLM** — Base URL / 모델 / API Key. 기본값은 OpenAI 호환 Chat(`gpt-4o-mini`).
3. **개인정보** — 비식별화 on/off, 마스킹할 환자명, 처리 후 녹음 폐기.
4. **용어 사전** — 다빈도 시술·질환·약품명(오인식이 잦은 것 위주).

설정값은 브라우저 `localStorage`(Electron은 앱 프로필)에 저장되며 서버로 전송되지 않습니다.

## 모드

- **진료실 · SOAP** — S(주관적)/O(객관적)/A(평가)/P(계획) 의무기록.
- **상담실 · 요약** — 관심 시술 / 안내 가격 / 상담 요약 / 다음 예약·후속 조치.

## ⚠ 규제 — 개발/도입 전 반드시 결정

환자 대화를 **해외 API로 전송**하는 구조는 개인정보보호법·의료법 컴플라이언스 이슈가 있습니다.
아래를 먼저 확정하세요.

- **개인정보 비식별화** — 위젯은 전송 전 텍스트에서 주민번호·전화번호·환자명을 마스킹합니다.
  단, **오디오 자체는 마스킹할 수 없습니다.** 민감한 경우 아래 리전 옵션을 쓰세요.
- **환자 동의 절차** — 녹음/처리에 대한 동의 방식.
- **리전 / 온프레미스** — 설정의 STT/LLM **Base URL**을 국내 리전 또는 자체 서버로 바꾸면
  해외 전송을 피할 수 있습니다(OpenAI 호환 인터페이스면 그대로 연동).

## MVP 범위 밖 (2단계)

EMR DB 직접 연동, 다중 사용자 계정, 클라우드 저장, 실시간 화자 분리 자동화.
